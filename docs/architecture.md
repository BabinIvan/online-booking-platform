# Архитектура системы

## Компоненты системы

| Компонент | Описание | Технология |
|-----------|---------|-----------|
| Frontend | Веб-интерфейс приложения | React/Vue.js |
| Backend | Сервер приложений | Node.js/Python |
| База данных | Хранилище данных | PostgreSQL |
| Платежная система | Обработка платежей | Stripe/PayPal |
| Модуль бронирования | Логика бронирования | Микросервис |

## Архитектурная диаграмма

```
┌─────────────────────┐
│   Пользователь     │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ Веб-интерфейс      │
│    (Frontend)       │
└──────────┬──────────┘
           │
           ▼
┌─────────────────────┐
│ Сервер приложений   │
│    (Backend)        │
└──┬────────┬────────┬┘
   │        │        │
   ▼        ▼        ▼
┌─────┐ ┌──────────┐ ┌─────────────┐
│ БД  │ │ Модуль   │ │  Платежная  │
│     │ │ брониров.│ │   система   │
└─────┘ └──────────┘ └─────────────┘
```

## Поток данных

| Этап | Действие | Откуда | Куда |
|------|----------|--------|------|
| 1 | Запрос | Frontend | Backend |
| 2 | Проверка | Backend | База данных |
| 3 | Бронирование | Backend | Модуль брониров. |
| 4 | Платеж | Backend | Платежная система |
| 5 | Ответ | Backend | Frontend |

## Основные API endpoints

| Метод | Endpoint | Описание |
|-------|----------|---------|
| GET | `/api/bookings` | Получить все бронирования |
| POST | `/api/bookings` | Создать новое бронирование |
| GET | `/api/bookings/:id` | Получить бронирование по ID |
| PUT | `/api/bookings/:id` | Обновить бронирование |
| DELETE | `/api/bookings/:id` | Удалить бронирование |
| POST | `/api/payments` | Обработать платеж |
